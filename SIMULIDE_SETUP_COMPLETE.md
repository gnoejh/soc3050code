## 🎯 Quick Start (Students)

### Method 1: One-Click Simulation (Recommended)
1. Open any project file (e.g., `projects/Port_Basic/Main.c`)
2. Press **`Ctrl+Shift+B`**
3. Select **"Build and Simulate Current Project"**
4. Click **Play (▶)** in SimulIDE when it opens
5. Done! 🎊

### Method 2: PowerShell Command
```powershell
.\cli-simulide.ps1 -ProjectDir "projects\Port_Basic"
```

### Method 3: Separate Build & Simulate
```powershell
# Build first
.\cli-build-project.ps1 -ProjectDir "projects\Port_Basic"

# Then simulate (skip rebuild)
.\cli-simulide.ps1 -ProjectDir "projects\Port_Basic" -BuildFirst $false
```

## 🔧 What Was Fixed

### Issue 1: Executable Path ✅
- **Problem:** Script looked for `bin\simulide.exe` (doesn't exist in portable version)
- **Solution:** Now checks `SimulIDE_1.1.0-SR1_Win64\simulide.exe` (root folder)

### Issue 2: PowerShell Compatibility ✅
- **Problem:** `GetRelativePath()` method not available in PowerShell 5.1
- **Solution:** Created custom `Get-RelativePath` function using `System.Uri`

### Issue 3: Circuit File Confusion ✅
- **Problem:** Two circuit files mentioned (`Simulator.simu` and `atmega128.simu`)
- **Solution:** Standardized on `Simulator.simu` only

### Issue 4: HEX File Not Loading ✅
- **Problem:** Relative paths weren't working reliably in SimulIDE
- **Solution:** Use **absolute paths** with forward slashes
  - Example: `Program="W:/soc3050code/projects/Port_Basic/Main.hex"`

## 🚀 How It Works

### Automated Workflow
1. **Build** → Compiles `Main.c` to `Main.hex`
2. **Update** → Creates temp circuit file with absolute HEX path
3. **Launch** → Opens SimulIDE with firmware pre-loaded
4. **Auto-Load** → Firmware loads automatically when you press Play

### Technical Details
- **Temporary circuit:** `%TEMP%\atmega128_temp.simu`
- **Original circuit:** `Simulator.simu` (unchanged)
- **HEX path format:** Absolute path with forward slashes
- **Auto_Load:** Enabled automatically in temp circuit

## 🎮 Using SimulIDE

### Basic Controls
- **Play (▶)** - Start simulation
- **Pause (⏸)** - Pause simulation
- **Stop (⏹)** - Stop and reset
- **Speed slider** - Adjust simulation speed
- **Ctrl+R** - Reset MCU

### Interacting with Circuit
- **LEDs** - Click to toggle (or watch them respond to program)
- **Buttons** - Click to press/release
- **Potentiometers** - Drag slider to adjust resistance
- **Serial Terminal** - Double-click `SerialTerm-1700` to open

### Pin Connections
```
PORTB (PB0-PB7) → 8 LEDs (with 220Ω resistors)
PORTD (PD0-PD7) → 8 Buttons (with 10kΩ pull-up resistors)
ADC0 (PF0) → Potentiometer (temperature sensor)
ADC1 (PF1) → Potentiometer (light sensor/CDS)
ADC2 (PF2) → Joystick X-axis
ADC3 (PF3) → Joystick Y-axis
```

## 🔍 Troubleshooting

### SimulIDE doesn't launch
```powershell
# Verify executable exists
Test-Path "W:\soc3050code\SimulIDE_1.1.0-SR1_Win64\simulide.exe"

# Try manual launch
Start-Process "W:\soc3050code\SimulIDE_1.1.0-SR1_Win64\simulide.exe"
```

### Firmware doesn't load
1. Right-click ATmega128 chip → Properties
2. Check "Program" path: Should be `W:/soc3050code/projects/Port_Basic/Main.hex`
3. Check "Auto_Load": Should be checked ☑
4. Manually load: Right-click → Load Firmware → Browse to HEX file

### Program doesn't run
1. Click **Play** button (▶)
2. Check MCU frequency: Should be **16 MHz**
3. Reset MCU: Press **Ctrl+R**
4. Check serial output: Double-click **SerialTerm-1700**

### Build fails
```powershell
# Clean and rebuild
cd projects\Port_Basic
Remove-Item *.o, *.elf, *.hex -ErrorAction SilentlyContinue
cd ..\..
.\cli-build-project.ps1 -ProjectDir "projects\Port_Basic"
```
